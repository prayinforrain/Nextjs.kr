Next.js는 루트 디렉터리의 `public` 폴더에서 이미지와 같은 정적 파일을 제공할 수 있습니다. `public` 내부의 파일은 기본 URL (`/`)로 시작하는 코드로 참조할 수 있습니다.

예를 들어 `public` 폴더에 `me.png`를 추가하면 다음 코드가 이미지에 접근합니다.

```jsx filename="Avatar.js"
import Image from 'next/image'

export function Avatar() {
  return <Image src="/me.png" alt="me" width="64" height="64" />
}
```

<PagesOnly>

이 폴더는 또한 `robots.txt`, `favicon.ico`, Google Site Verification, (`.html`을 포함하는) 기타 정적 파일들에 유용합니다.
하지만 `pages/` 디렉터리에 있는 파일과 동일한 이름을 가진 정적 파일이 있으면 에러가 발생할 수 있습니다. [더보기](/docs/messages/conflicting-public-file-page).

</PagesOnly>

<AppOnly>

`robots.txt`, `favicon.ico` 등과 같은 정적 메타데이터 파일의 경우 `app` 폴더에 있는 [special metadata files](/docs/app/api-reference/file-conventions/metadata) 를 사용해야 합니다.

</AppOnly>

> 유용한 정보:
>
> - 디렉터리 이름은 `public`이어야 합니다. 이름은 바꿀 수 없으며 정적 데이터를 제공하는 유일한 디렉터리입니다.
> - [빌드 타임](/docs/app/api-reference/next-cli#build)에 `public` 디렉터리에 있는 데이터만 Next.js 에서 제공할 것 입니다. 런타임에 추가된 파일은 사용할 수 없습니다. 영구 파일 저장은 [AWS S3](https://aws.amazon.com/s3/)와 같은 서드파티 서비스를 추천합니다.
